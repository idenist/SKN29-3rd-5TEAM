## 1. 테스트 개요

### 3.4 실패 케이스 분석

**TC002 (2019년 공고 제외):** 사용자가 "2019년 것 말고"라고 명시하였음에도 답변 텍스트에 "2019" 키워드가 포함되었다. Answer Generator 단계에서 금지 키워드 필터링 로직이 없어 발생한 문제이다.

**TC003 (화성 우주창업):** 내부 검색 결과가 부족한 경우 외부 공식 출처 검색으로 분기하는 ReAct fallback 기능이 현재 workflow에 구현되어 있지 않다. `internal_search_sufficient`, `next_action`, `external_search_status`, `external_search_targets` 필드가 응답에 존재하지 않아 관련 항목이 전부 실패하였다.

**TC010 (빈 질문):** 빈 질문 입력 시 FastAPI가 422를 반환하는 것이 기대 동작이다. 초기 평가 스크립트가 422를 예외로 처리하여 score=0으로 집계되었으나, 스크립트 수정 후 정상 통과(1.000)로 처리된다.

## 5. 한계점 분석

### 5.1 ReAct Fallback 미구현

내부 검색 결과가 부족하거나 모두 마감된 경우 외부 공식 출처로 분기하는 Result Sufficiency Checker 및 ReAct fallback이 구현되어 있지 않다. 이로 인해 TC003과 같이 관련 정책이 없는 질문에도 유사도 기반으로 관련성이 낮은 정책을 추천하는 문제가 발생한다.

**개선 방향:** LangGraph workflow에 Result Sufficiency Checker 노드 추가 및 external_search 분기 구현.

### 5.2 금지 키워드 필터링 부재

사용자가 특정 연도나 항목을 제외하도록 요청한 경우에도 Answer Generator가 해당 키워드가 포함된 텍스트를 그대로 답변에 사용한다.

**개선 방향:** Answer Generator 단계에서 사용자 질문의 부정 표현을 파싱하여 금지 키워드 필터링 적용.

### 5.3 source_category 라우팅 실패

직업훈련 특화 키워드(지게차, 국비지원, 자격증 등)에 대해 training source_category 대신 policy 도메인으로 라우팅되는 경우가 발생한다.

**개선 방향:** 직업훈련, 국비지원, 자격증 키워드에 대해 training source_category 가중치 강화.

### 5.4 신청기간/제출서류 정보 부족

대부분의 정책에서 신청기간, 신청방법, 제출서류 정보가 "제공된 데이터에는 정보가 없습니다"로 출력된다. summary chunk 중심 검색으로 인해 application chunk가 함께 반환되지 않는 구조적 한계이다.

**개선 방향:** 동일 item_id의 summary + application chunk를 함께 조회하는 multi-chunk 검색 구현.

### 5.5 레이턴시

질문당 평균 응답 시간이 10~20초로 실시간 서비스에서 체감 지연이 있다. OpenAI API 호출이 Condition Extractor, Router, Answer Generator 총 3회 발생하는 구조가 주요 원인이다.

**개선 방향:** 비동기 처리 적용, 또는 경량 모델(gpt-4o-mini)로 Condition Extractor/Router 대체.

## 8. 결론

본 시스템은 두 가지 평가 방법으로 검증하였다.

규칙 기반 자동 평가에서는 10케이스 중 8건 통과(통과율 80%, 평균 점수 0.893)를 기록하였다. 실패 케이스 중 TC002(금지 키워드 필터링)와 TC003(ReAct fallback 미구현)은 향후 개선이 필요한 항목으로 식별되었다.

LLM-as-a-Judge 평가에서는 Context Relevance 4.20, Groundedness 4.10, Answer Relevance 4.00으로 세 지표 모두 4점 이상의 우수한 품질을 확인하였다. 특히 주거, 창업, 일경험 도메인에서 높은 관련성과 근거성을 보였다. 직업훈련 특화 키워드에 대한 라우팅 정확도는 향후 개선이 필요한 항목으로 식별되었다.
