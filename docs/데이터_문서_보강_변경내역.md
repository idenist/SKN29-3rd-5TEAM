## 2. 전체 보강 방향

기존 문서의 큰 구조와 서술 흐름은 유지하고, 실제 스크립트와 비교했을 때 누락될 수 있는 보조 설명과 산출물 목록만 추가했다. 특히 수집 placeholder 스크립트의 현재 상태, K-Startup/HRD 전처리 산출물, 최종 통합 검증 리포트, FastText 보강 스크립트, 청킹 단계 구분을 보완했다.

## 4. 크게 바꾸지 않은 내용

다음 내용은 기존 문서의 방향을 유지했다.

- 공식 데이터 출처 3종: 온통청년, K-Startup, 고용24/HRD
- 최종 통합 기준: 온통청년 전체 + K-Startup HIGH + HRD HIGH
- 최종 통합 데이터 수량: 26,803건
- 최종 RAG 청크 수: 33,950건
- Ground Truth 평가 질문 수: 50건
- `item_id` 중심 연결 원칙
- 원본 데이터 미수정 원칙
- 결측값 임의 보정 금지 원칙
- `info_score`는 추천 점수가 아닌 데이터 완성도 점수라는 설명
- MEDIUM/LOW 데이터는 최종 통합하지 않고 참고용으로 보관한다는 기준

## 5. 발표 시 강조하면 좋은 보강 포인트

- “수집 코드를 모두 자동화했다”가 아니라, MVP에서는 확보된 공식 raw 데이터를 기준으로 전처리와 통합 품질을 우선했고, 재수집/상세 크롤링은 확장용 스크립트로 분리했다고 설명하는 것이 안전하다.
- 평가 기준 대응 측면에서는 KoNLPy, 불용어, BoW, TF-IDF, Word2Vec/FastText를 실제 서비스 검색에 직접 쓰는 것이 아니라, 데이터 이해 및 평가 대응용 리포트로 사용했다고 설명하면 된다.
- RAG 검색 기준은 최종 통합 청크인 `opportunity_chunks.jsonl`이며, 온통청년 `chunks.jsonl`은 정책 데이터 1차 전처리 산출물로 구분해서 설명하면 된다.
